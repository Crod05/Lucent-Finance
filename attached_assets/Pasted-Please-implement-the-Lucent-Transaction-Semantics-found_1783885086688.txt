PHASE 3 — Centralized semantics evaluator
--------------------------------------------------

Create:

artifacts/api-server/src/lib/transaction-semantics.ts

Implement one centralized evaluator:

evaluateTransactionSemantics()

This must become the single source of truth for:

- budgets
- dashboard totals
- spending insights
- Budget Guardian
- quests
- chapters
- scorecards
- future boss battles

The evaluator must be:

- pure
- deterministic
- free of database queries

It receives semantic facts as input and returns evaluator outputs.

Implement the financial rules defined in:

docs/transaction-semantics.md

including:

- transfers never create spending or income
- refunds reduce spending
- reimbursements reduce allocated personal spending
- debt proceeds are never income
- investment contributions are not consumption spending
- ambiguous transactions produce no evidence
- unclassified transactions cannot support gamification

Do not duplicate semantic logic elsewhere.
